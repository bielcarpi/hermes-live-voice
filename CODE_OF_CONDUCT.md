# Code of Conduct

Hermes Live Voice should be a practical, welcoming place to build open-source voice infrastructure.

## Expected behavior

- Be direct, kind, and specific.
- Critique ideas and code, not people.
- Respect different experience levels, languages, identities, and deployment constraints.
- Share enough evidence for others to reproduce a claim.
- Protect user privacy, credentials, audio, prompts, and agent output.
- Accept maintainer decisions about scope, safety, and reviewability even when a proposal is declined.

## Unacceptable behavior

- Harassment, discrimination, threats, sexualized attention, or personal attacks.
- Doxxing or publishing private correspondence, credentials, audio, prompts, logs, or identifying information without permission.
- Repeated bad-faith argument, spam, promotional flooding, or deliberately derailing technical discussion.
- Pressuring contributors to expose private deployments or provider accounts.
- Testing vulnerabilities against systems you do not own or have permission to assess.

## Scope

This code applies to repository issues, pull requests, discussions, reviews, release channels, and public spaces where someone is representing the project.

## Enforcement

Maintainers may edit or remove contributions, lock conversations, issue warnings, temporarily restrict participation, or permanently ban participants when necessary to protect the community and project.

For an urgent conduct report, contact the maintainer privately through the GitHub profile. Do not disclose personal or security-sensitive details in a public issue. Security vulnerabilities should use the separate process in [SECURITY.md](SECURITY.md).

Reports will be reviewed as privately and promptly as practical. Retaliation against someone who makes a good-faith report is itself a violation.
